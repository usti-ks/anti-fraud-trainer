# Антифрод‑тренажёр

Проект подготовлен для безопасного подключения GigaChat и хранения в GitHub без утечки ключа.

## Что изменено

- GigaChat подключается **только через серверный прокси**: `server/gigachat-proxy.mjs`
- фронтенд по-прежнему обращается только к `/api/gigachat`
- ключ больше не нужно вставлять в `src/`, `public/`, `index.html` или `VITE_*`
- мобильное burger menu переделано: overlay, блокировка скролла, закрытие по клику вне меню, по пункту меню и по `Esc`
- добавлены `.gitignore` и `.env.example` для безопасной загрузки на GitHub

## Куда конкретно вставлять ключ

Создайте в **корне проекта** файл `.env.local` рядом с `package.json` и вставьте туда:

```env
GIGACHAT_AUTH_KEY=ВАШ_РЕАЛЬНЫЙ_КЛЮЧ_GIGACHAT
GIGACHAT_PORT=8787
```

Именно сюда вставляется ключ локально.

### На хостинге

Если будете выкладывать не просто код в GitHub, а запускать проект с рабочим GigaChat, то тот же ключ нужно добавить в **Environment Variables / Secrets** хостинга под именем:

```env
GIGACHAT_AUTH_KEY
```

## Где этот ключ читается в коде

Файл:

- `server/gigachat-proxy.mjs`

Строка логики теперь такая:

- `process.env.GIGACHAT_AUTH_KEY`

То есть ключ берётся только из окружения.

## Запуск локально

Установка:

```bash
npm install
```

Только фронтенд:

```bash
npm run dev:web
```

Фронтенд + локальный прокси GigaChat:

```bash
npm run dev:full
```

Отдельно только прокси:

```bash
npm run proxy
```

## Сборка

```bash
npm run build
```

## Загрузка на GitHub

В репозиторий можно загружать проект как обычный кодовый репозиторий.

В GitHub **не должны попадать**:

- `.env.local`
- `.env`
- любые файлы с реальным ключом
- `node_modules`
- `dist`

Это уже закрыто в `.gitignore`.

## Важное ограничение

Если разместить проект на **GitHub Pages**, безопасный GigaChat работать не будет, потому что Pages — это статический хостинг без Node.js-прокси.

Для рабочего GigaChat нужен хостинг, где можно запустить backend / Node.js.
